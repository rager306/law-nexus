# M044 ACP Default Inclusion Policy

**Date:** 2026-05-22

## Status

Accepted policy for `M044-2i004p / S01`.

This policy defines which ACP governance records may enter default canonical architecture extraction and which record meanings remain blocked.

## Decision

M044 may integrate the existing minimal ACP governance chain into default canonical architecture extraction, but only as bounded architecture-governance records.

Approved default inclusion scope:

- `prompt_record` as provenance only;
- `proposal` as proposed governance context only;
- `decision_candidate` as a candidate requiring authority;
- `proof_gate` as a gate definition only;
- `health_finding` as a diagnostic or blocker only;
- ACP governance edges that connect those records without implying accepted decisions or satisfied proof gates.

Default inclusion is allowed only if the extractor generates these rows from ACP source records or ACP contract/decision source mappings. It must not copy rows from ACP derived JSONL files into the default registry.

## Approved row classes

| ACP class | Default inclusion | Required constraints |
| --- | --- | --- |
| `prompt_record` | Allowed | Must remain provenance/context. It must not validate implementation, decisions, legal correctness, parser quality, or retrieval quality. |
| `proposal` | Allowed | Must remain proposed. It must not become an accepted decision or requirement. |
| `decision_candidate` | Allowed | Must keep `authority_required: true` and candidate/proposed status. It must not become a `decision` record. |
| `proof_gate` | Allowed | Must represent gate definition only. It must not imply proof satisfaction. |
| `health_finding` | Allowed | Must remain blocker/diagnostic. It must not imply readiness. |
| ACP governance edges | Allowed | Must preserve provenance/proposal/gate relationships without `satisfies`, `validated_by`, or other proof-positive semantics. |

## Blocked row meanings

Default extraction must not introduce:

- accepted ACP `decision` rows from `decision_candidate` records;
- proof-satisfaction rows from `proof_gate` definitions;
- requirement validation rows for R035, R037, or R038;
- product readiness, legal correctness, parser completeness, FalkorDB ingestion/runtime, graph-vector retrieval quality, or independent external review claims;
- any row whose only source anchor is an ACP derived JSONL file.

## Extractor ownership

Selected owner: `scripts/extract-prd-architecture-items.py`.

M044 S02 may extend the default extractor because M043 already proved opt-in architecture-build compatibility. The extension must be source-mapped and deterministic. It must preserve default `--check` currentness semantics.

Allowed implementation approach:

1. add ACP source paths to extractor-required source inventory;
2. add curated ACP item/edge builders that derive the five item rows and seven edge rows from tracked ACP source records and ACP contract/decision documents;
3. regenerate `prd/architecture/architecture_items.jsonl` and `prd/architecture/architecture_edges.jsonl` through the extractor command;
4. update tests to prove generated default ACP rows, exclusions, non-claims, authority requirement, and graph/verifier compatibility.

## Required source anchors

Each default ACP row must anchor to at least one of:

- ACP Markdown source records under `prd/architecture/acp/fixtures/`;
- ACP contracts/decisions under `prd/architecture/acp/`;
- ACP scripts/tests under `scripts/` or `tests/` only when the row describes tooling behavior.

Source anchors must be repository-relative and must not use:

- local absolute paths;
- transient execution-log paths;
- external URLs as proof anchors;
- raw provider payloads;
- raw vectors;
- raw legal text;
- ACP derived JSONL as sole source evidence.

## S02 implementation contract

S02 must prove all of the following:

- default extractor output includes exactly the approved ACP item and edge classes;
- no ACP row is copied from `canonical-projection`, `integrated-registry`, `composed-registry`, or `canonical-integration` JSONL;
- default canonical JSONL is regenerated by `uv run python scripts/extract-prd-architecture-items.py`;
- `uv run python scripts/extract-prd-architecture-items.py --check` passes after regeneration;
- `uv run python scripts/build-architecture-graph.py --check` passes after graph rebuild;
- `uv run python scripts/verify-architecture-graph.py` passes;
- tests cover ACP rows, excluded proof-positive semantics, source anchors, non-claims, authority requirement, and row counts.

Expected default count change if all approved records are included:

- items: from 58 to 63;
- edges: from 91 to 98.

## Verification obligations

M044 S02 must run:

```text
uv run python scripts/extract-prd-architecture-items.py
uv run python scripts/extract-prd-architecture-items.py --check
uv run python scripts/build-architecture-graph.py
uv run python scripts/build-architecture-graph.py --check
uv run python scripts/verify-architecture-graph.py
```

It must also run focused ACP extractor tests and existing ACP/projection tests.

## Required non-claims

Default inclusion does not validate:

- R035;
- R037;
- R038;
- parser completeness;
- legal correctness;
- FalkorDB ingestion or runtime loading;
- graph-vector retrieval quality;
- production readiness;
- independent external review.

## Blocked actions

- Do not hand-edit `prd/architecture/architecture_items.jsonl`.
- Do not hand-edit `prd/architecture/architecture_edges.jsonl`.
- Do not copy ACP derived JSONL into default canonical JSONL.
- Do not use opt-in integration success alone as default currentness proof.
- Do not promote ACP decision candidates to accepted decisions.
- Do not treat proof-gate definitions as proof-gate satisfaction.
- Do not update R035, R037, or R038 to validated status from M044 evidence.
- Do not initialize git-lex, RDF, SHACL, SPARQL, or dashboard authority layers in M044.

## S03 decision input

S03 should decide among:

1. adopt default ACP inclusion for the approved governance rows;
2. adopt only a subset if S02 exposes verifier or claim-safety concerns;
3. revert to opt-in integration if default inclusion weakens source-truth or verifier clarity.

Default recommendation: adopt default inclusion only if S02 produces extractor-generated rows, passing tests, and a green default verifier.
