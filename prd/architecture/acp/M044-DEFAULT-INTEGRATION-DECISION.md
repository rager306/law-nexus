# M044 Default ACP Canonical Extractor Integration Decision

## Status

Adopted for the current bounded ACP fixture scope.

## Decision

The default architecture extractor may include the approved ACP governance rows in the tracked canonical architecture registry, provided those rows are generated by the extractor from explicit source mappings and remain bounded as non-authoritative architecture-governance records.

For M044, the approved default inclusion set is:

- `ACP-APR-0001` as `prompt_record`
- `ACP-AP-0001` as `proposal`
- `ACP-DC-0001` as `decision_candidate`
- `ACP-PG-0001` as `proof_gate`
- `ACP-AHF-0001` as `health_finding`
- the seven governance edges linking those records through prompt/proposal/decision/proof/health relationships

The default canonical registry now has 63 items and 98 edges. The default architecture graph and verifier accept that registry with zero verifier failures.

## Source ownership

`scripts/extract-prd-architecture-items.py` is the owner for these default ACP rows. It generates the rows directly from tracked ACP source artifacts and source anchors.

The following remain blocked:

- hand-editing `prd/architecture/architecture_items.jsonl`
- hand-editing `prd/architecture/architecture_edges.jsonl`
- copying `prd/architecture/acp/derived/*.jsonl` into canonical registry files
- promoting a `decision_candidate` into an accepted architecture decision without an explicit future decision workflow
- treating a proof-gate record as proof-gate satisfaction

## Compatibility with previous ACP proof paths

The prior custom and opt-in ACP composition tools remain valid as diagnostic compatibility surfaces, but their role changes after default inclusion:

- they validate supplied ACP projection inputs;
- they skip ACP rows whose IDs are already present in the default canonical registry;
- their generated custom outputs now remain at 63 items and 98 edges for the current fixture set;
- their reports expose added vs already-integrated ACP counts.

This preserves the M041/M042/M043 proof paths as idempotent non-authoritative checks rather than duplicate appenders.
